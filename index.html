Praxio: Importar Planejamento

Utilização: Carregar arquivo xlsx pre-formatado e configurado, selecionar tipo de exportação (txt ou csv)

Dica: ALT+K exibe atalhos disponíveis na página

Atenção: Antes de importar é necessário ter configurado previamente o módulo de escala no Globus em Utilitários / Configurações / Configurações de Importação

Configurações da engine:

Atenção: Alterar estes arquivos pode corromper a lógica do app, ajuste com cautela.

Arquivos do motor de conversão
File Description
settings.js Arquivo principal de configurações, ver detalhes mais abaixo
engine.js Motor de processamento
export.js Exportador de arquivo
settings.js
guiaAnalise:
Nome da "guia" do arquivo a ser tratada pelo engine
atividades:
Códigos do módulo de escala para tipo de viagem (produtiva, troca turno, recolhe)
cells:
Células de referência na planilha para dados de importação (nome planejamento, cod linha, tempos de acesso / recolhida, etc)
layout:
Dicionário com layout definido para arquivos de importação
Cada entrada no array de layout representa um dado no arquivo de exportação, ex:
{ field: "COD_PROG", size: 7, pad: " ", align: "L", resolve: (ctx) => ctx.global.codProg }
field:
Nome no campo (csv)
size:
Tamanho mínimo (caracteres) a ser ocupado nos arquivos de exportação
pad:
String a ser usada para completar retorno compativel com tamanho definido
align:
Define se complemento será aplicado no inicio R ou no fim L do retorno
resolve:
Função (ou string) a ser usado no campo
Resolve pode fazer referência a uma entrada de contexto do engine ctx.suaFuncao ou uma função simples, exemplos:
Exemplo 1 - Conteúdo Fixo
{ field: "FOO", size: 3, pad: "0", align: "L", resolve: (ctx) => "BAR" }
Exemplo 2 - Função de contexto
{ field: "FEI", size: 3, pad: "0", align: "L", resolve: (ctx) => ctx.global.feiFunction }
Neste caso deve ser criado a função feiFunction no arquivo de engine.js
