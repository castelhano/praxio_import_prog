Importação de Planejamento - Globus

Guia para preenchimento da planilha, configuração e entendimento do motor de processamento de programações de transporte urbano.

👤 Usuário Excel ⚙️ Técnico / Dev 📊 Planilha de Dados

Visão Geral do Sistema

Este sistema lê uma planilha Excel com os horários de viagens de uma linha de ônibus, processa as escalas por carro e gera um arquivo de saída TXT posicional ou CSV pronto para importação no sistema de gestão.

1

Preencher a Planilha

O usuário preenche a aba dados com parâmetros gerais, grade de viagens, trocas de turno e eventuais exceções.

2

Carregar arquivo e definir exportação

O arquivo .xlsx é carregado na interface web, selecione o tipo de exportação (TXT, CSV) e precione Processar

3

Processamento

O Engine percorre cada carro da grade, monta viagens, divide em tabelas, nomeia e aplica exceções.

4

Exportar

O resultado é baixado como TXT (largura fixa) ou CSV separado por ponto-e-vírgula.

ℹ️
Por padrão, o Engine busca na planilha uma aba chamada dados, esse nome pode ser configurado em settings.guiaAnalise, toda a lógica de leitura parte dessa aba.
Parte 1 Guia do Usuário Excel

Parâmetros Gerais (Células F2–F11)

As células da coluna F (configuráveis em settings.cells) definem os parâmetros globais da escala. Elas ficam por padrão no canto superior esquerdo da aba dados.

Célula Parâmetro Exemplo Descrição
F2 Código de Programação 410U07 Nome da programação criada no Globus.
F3 Código da Linha 410 Número da linha de ônibus.
F4 Código de Localidade — Ida 20 Código do local para o ponto 1, usado para o codigo das viagens de IDA.
F5 Código de Localidade — Volta 61 Código do local para o ponto 2, usado para o codigo das viagens de VOLTA.
F6 Tempo de Preparo (min) 10 Minutos de preparo antes da primeira viagem de cada tabela.
F7 Tempo de Acesso (min) 15 Minutos de acesso (saída da garagem). Subtraído do horário da 1ª viagem para calcular saída de garagem.
F8 Tempo de Recolhe (min) 20 Minutos somados ao fim da última viagem para calcular o horário da entrada de recolhe.
F9 Código Local de Pegada 51 Código de local da garagem.
F10 É Circular? 1 Digite true, sim ou 1 para ativar modo de linha circular. Qualquer outro valor assume comportamento padrão.
F11 Layout de Colunas 1 Define quantas colunas ocupa cada carro na grade. Veja a seção Layouts de Colunas.
💡
Os tempos em F6, F7 e F8 podem ser inseridos como número inteiro (minutos) ou no formato HH:MM. Por exemplo, 10 ou 00:10 são equivalentes.

Grade de Viagens

A grade principal fica no intervalo I4:AV33 (configurável). As primeiras 3 linhas da grade têm funções especiais:

Linha na Grade Linha Excel (padrão) Conteúdo
Linha 1 Linha 1 do Excel Número de sequência do carro (1, 2, 3…). Identifica cada bloco de colunas.
Linha 2 Linha 2 do Excel Sentido da coluna: I (ida) ou V (volta) — apenas informativo.
Linha 3 Linha 3 do Excel Código de localidade usado para alterar todas as viagens de um sentido do carro.
ℹ️ Layout das viagens: O Engine permite utilização de multiplos layouts para entrada de dados. Por padrão são criado os layouts:
  1. 2 colunas (Ida / Volta): Layout base, cada carro ocupa duas colunas no grid sendo a primeira as viagens de IDA e a segunda as viagens de VOLTA
  2. 3 colunas (Ida / Volta / Fim volta): Neste layout são apresentado 3 linhas por carro, o que varia do layout 1 é a adição da terceira coluna com horário de fim das viagens de volta, deve ser informado manualmente.
  3. 4 colunas (Ida / ref / Volta / ref): Layout com uma coluna de referência entre as viagens de IDA e VOLTA, estas colunas são ignoradas no Engine, mantendo o comportamento que o fim da viagem de IDA eé o inicio do VOLTA, e vice versa.

Exemplo de Grade — Layout 1 (2 colunas por carro)

Cada carro ocupa 2 colunas: Ida e Volta, onde o fim da viagem de IDA é o inicio do VOLTA, mesma regra se aplica para o fim da viagem de volta.

2 sentido: IV IV IV
3 local: 12

⚠️
Textos especiais na grade: palavras como INTERV, RECO, TT ou qualquer texto que não seja um horário válido são ignorados pelo engine. Eles servem apenas como anotação visual para o usuário. O que realmente sinaliza um intervalo ou recolhe é a ausência de um horário válido na célula de fim de viagem.

Como o engine calcula o fim de cada viagem

  • Via de regra, e isso se aplica a todos os layouts, a lógica é que uma viagem de IDA é considerada Produtiva se existe uma horário válido de fim na coluna imediatamente a frente dela.
  • Para o volta o engine busca se existe uma viagem de fim válida na linha abaixo na coluna anterior, ou seja se existe um lançamento na coluna de IDA.
  • Em ambos os casos, se não é identificado fim para viagem, o engine trata o lançamento como Intervalo ou Recolhida, se existe viagens posteriores considera Intervalo, se não trata como Recolhida.
  • É possivel configurar no arquivo settings.js se será gerado uma entrada para estes casos de intervalo e recolhe, e como deve ser o comportamento deles (detalhado melhor na seção técnica, subitem de comportamento)

Trocas de Turno (TT)

As linhas de troca de turno ficam em geral ao final da grade de horários, por padrão é disponibilizado campo para 3 trocas de turno por carro, ou sejá até 4 condutores por veículo.

Cada troca de turno deve ser colocada no seu respectivo sentido e deve corresponder a uma viagem válida do grid, se informado troca de turno 14:30 na coluna referente a ida, o engine ira encerrar a tabela se conseguir localizar essa viagem no sentido informado, se não ignora entrada.

1 seq: 11 22 33
2 sentido: IV IV IV
...
34 TT 1o: 12:10— —16:00 13:15—
35 TT 2o: —16:00 —— ——
36 TT 3o: —— —— ——
💡
O horário de TT deve corresponder exatamente a um horário existente na grade de viagens daquele carro. Se não houver correspondência, o TT é ignorado.

O sentido do TT é determinado pela coluna onde o horário foi inserido (Ida → sentido I, Volta → sentido V).

Exceções de Viagem

Permitem sobrescrever a atividade, o local ou a linha (caso viagem pertença a outra linha) de viagens específicas, ou várias (alteração em lote). Ficam no intervalo A40:G60 (configurável).

Coluna Campo Tipo Descrição
A Carro Critério Número do carro (ex: 1 ou 01). Deixar vazio = aplica a todos.
B Viagem (horário) Critério Horário de início da viagem (ex: 13:49). Deixar vazio = aplica a qualquer horário.
C Sentido Critério I, V ou C. Deixar vazio = qualquer sentido.
D Tipo Critério P = produtiva, O = ociosa (TT, intervalo, recolhe). Deixar vazio = ambos.
E Atividade Override Código da atividade a usar no lugar do padrão (ex: 03).
F Local Override Código de localidade alternativo.
G Linha Override Código de linha alternativo (preenche o campo COD_LINHA na saída).

Exemplo

Para que o carro 2, na viagem das 13:10 sentido I, use a atividade 03 e o local 99:

A (Carro)B (Viagem)C (Sentido)D (Tipo)E (Atividade)F (Local)G (Linha)
2 13:10 I P 03 99 —
⚠️
As regras são aplicadas em sequência e acumuladas (merge). Se duas regras conflitarem, a última da lista prevalece para cada campo.

Exceções de Tabela

Permitem sobrescrever configurações de tabelas inteiras (nome, horários, tempos, saída de garagem). Ficam no intervalo L40:T60.

Coluna Campo Exemplo Descrição
L Tabela (chave) 01A Nome da tabela a sobrescrever (carro + letra, ex: 01A, 02B).
M Nome 01Z Substitui o nome gerado automaticamente pelo engine.
N Início 05:00 Sobrescreve o horário de início da tabela (INICIO_SERVICO).
O Fim 12:10 Sobrescreve o horário de fim da tabela (FIM_SERVICO).
P Período (Turno) 1 Força o número do turno da tabela, ignorando o mapa automático.
Q Saída Garagem 04:45 Sobrescreve o horário de saída da garagem (SAIDA_GAR).
R Preparo (min) 15 Sobrescreve o tempo de preparo apenas para esta tabela.
S Acesso (min) 20 Sobrescreve o tempo de acesso apenas para esta tabela.
T Recolhe (min) 25 Sobrescreve o tempo de recolhe apenas para esta tabela.
💡
A chave da tabela (coluna L) é formada pelo número do carro com 2 dígitos + letra (ex: 01A, 03C). A busca é feita sem distinção de maiúsculas/minúsculas.

Layouts de Colunas (F11)

O valor de F11 define como a grade está organizada — quantas colunas cada carro ocupa e qual papel cada uma tem. Existem 4 layouts disponíveis:

Layout 1 — Padrão

2 colunas: Ida / Volta

O fim da Ida é o início da Volta. O fim da Volta é o início da próxima Ida.

Col 0 — ida_inicio Col 1 — volta_inicio
Layout 2

3 colunas: Ida / Volta / Fim Volta

O fim da Volta é explícito na terceira coluna.

Col 0 — ida_inicio Col 1 — volta_inicio Col 2 — volta_fim
Layout 3

4 colunas (colunas 1 e 3 ignoradas)

Estrutura de 4 colunas onde apenas col 0 e col 2 são lidas. Útil quando há colunas auxiliares na planilha.

Col 0 — ida_inicio Col 1 — ignorada Col 2 — volta_inicio Col 3 — ignorada
Layout 4

4 colunas com fim explícito em ambos

Fim de Ida e Fim de Volta ambos explícitos. Nenhum fim é inferido da linha seguinte.

Col 0 — ida_inicio Col 1 — ida_fim Col 2 — volta_inicio Col 3 — volta_fim
ℹ️
Se a célula F11 estiver vazia ou com um valor não reconhecido, o engine usa automaticamente o Layout 1 como padrão.

Linha Circular (F10 = 1)

Quando a linha é circular, os ônibus percorrem o trajeto completo em laço contínuo, sem distinguir Ida e Volta da mesma forma que uma linha convencional.

Comportamento

Com F10 = 1, o engine ativa o modo circular com as seguintes regras:

🔄
Viagem normal (só col Ida preenchida): sentido C, fim = próxima Ida.

Meia viagem IDA (cols Ida e Volta preenchidas na mesma linha): o carro inicia no meio do percurso. Sentido conforme circularMeiaViagemConsisteSentido.

Meia viagem VOLTA (só col Volta preenchida, logo após intervalo): o carro retoma no meio do percurso.
💡
As configurações circularGeraMeiaViagem e circularMeiaViagemConsisteSentido no settings.js controlam se a meia viagem é emitida na saída e qual sentido (I/V vs. C) ela recebe.
Parte 2 Referência Técnica

Arquitetura do Sistema

O sistema é composto por três módulos JavaScript independentes que se comunicam de forma unidirecional:

S

settings.js — Configuração

Expõe o objeto global SETTINGS com todos os parâmetros de comportamento: mapeamento de células, layouts, atividades, sequências de letras e layout de saída. É o único arquivo que deve ser editado para customizar o comportamento do sistema.

E

engine.js — Motor de Processamento

Expõe o objeto global Engine. Recebe um workbook SheetJS e retorna um array de contextos { trip, global }. Não conhece a camada de exportação.

X

export.js — Exportador

Expõe o objeto global Exporter. Recebe o array do Engine e gera TXT posicional ou CSV. Toda a formatação de campo é guiada por SETTINGS.layout.

settings.js — Referência Completa

Todas as propriedades do objeto SETTINGS, seus tipos, valores padrão e impacto no sistema.

Flags de Comportamento

guiaAnalise string
Nome da aba do Excel a ser processada.

O engine busca workbook.Sheets[SETTINGS.guiaAnalise]. Se a aba não existir, o processamento falhará silenciosamente.

Padrão: 'dados'
geraEntradaTrocaTurno boolean
Controla se a linha de encerramento com atividade troca_turno é emitida.

Quando true, ao fechar uma tabela por TT, o engine emite uma linha extra com a atividade configurada em atividades.troca_turno.

Padrão: true
geraEntradaIntervalo boolean
Controla se a linha de encerramento com atividade intervalo é emitida.

Quando true, ao fechar uma tabela por intervalo (viagem sem fim), o engine emite uma linha extra com a atividade de intervalo.

Padrão: true
geraEntradaRecolhidas boolean
Controla se a linha de recolhe é emitida ao fim da última tabela do carro.

O horário de chegada da entrada de recolhe é calculado como fim_última_viagem + recoMins.

Padrão: true
aplicarFormatacaoNoCsv boolean
Se true, aplica padding/alinhamento também no CSV.

Quando false, o CSV contém os valores brutos sem padding. Útil quando o sistema receptor do CSV faz sua própria formatação.

Padrão: true
primeiroCodViagem number
Primeiro valor do contador sequencial de viagens (COD_VIAGENS).

O sequencial começa neste valor para cada tabela e é incrementado a cada linha emitida, incluindo as linhas de encerramento (TT, intervalo, recolhe).

Padrão: 10

Mapa de Turnos

mapaTurnos object
Define o número do turno com base no horário de início da tabela.

As chaves são horários de corte: se tabStart < corte, aquele turno é atribuído. A última entrada deve ter um horário sentinela alto ('99:00') para capturar os demais.

mapaTurnos: {
  '12:00': 1, // início antes de 12:00 → turno 1
  '19:00': 2, // início entre 12:00 e 18:59 → turno 2
  '99:00': 4, // qualquer horário restante → turno 4
}

Atividades

atividades object
Códigos e locais padrão para cada tipo de entrada gerada.
produtiva
cod: '01' · local: null

Viagem normal em linha. O local é resolvido pelo sentido (codIda/codVolta) ou exceção.

intervalo
cod: '07' · local: '07'

Entrada de encerramento quando a tabela termina por intervalo.

troca_turno
cod: '10' · local: '10'

Entrada de encerramento quando a tabela termina por TT.

recolhe
cod: '11' · local: '11'

Entrada de encerramento da última tabela do carro.

Cada atividade pode ser sobrescrita por uma exceção de viagem (campo E da tabela de exceções).

Sequências de Nomeação de Tabelas

sequenciaNormal array
Letras usadas para nomear tabelas em sequência normal.

A primeira tabela de cada carro sempre recebe a primeira letra ('A'). As seguintes recebem a próxima letra disponível.

sequenciaNormal: ['A', 'B', 'D', 'E', 'F']
sequenciaPosIntervalo: ['C', 'V', 'X', 'Z']

Tabelas abertas após um intervalo usam sequenciaPosIntervalo. Por exemplo: carro 01 com um intervalo geraria tabelas 01A → 01C.

Layout de Saída (layout[])

layout array
Define os campos, tamanhos, alinhamentos e funções de resolução de cada coluna na saída.

Cada entrada do array define um campo da saída. O Exporter itera sobre este array para montar cada linha.

  align: 'L', // 'L' = esquerda, 'R' = direita

layoutColunas

layoutColunas object
Mapa de configurações de colunas da grade, indexado por chave (valor de F11).

Cada entrada define totalColunas (número de colunas por carro) e slots (mapeamento de papel → índice relativo 0-based dentro do bloco).

'4': {
  totalColunas: 4,
  slots: {
    ida_inicio: 0, // 1ª coluna do bloco
    ida_fim: 1, // fim explícito da ida
    volta_inicio: 2, // início da volta
    volta_fim: 3, // fim explícito da volta
  }
}

Para adicionar um novo layout, basta inserir uma nova chave neste objeto e informar o número correspondente em F11.

viagensConf

viagensConf object
Coordenadas dos intervalos da grade de viagens e das tabelas de exceções.
intervaloGeral: 'I4:AV33', // grade principal de viagens
intervaloExcecoesViagens: 'A40:G60', // tabela de exceções de viagem
intervaloExcecoesTabelas: 'L40:T60', // tabela de exceções de tabela

linhaSentido: 2, // linha 2 = sentido (informativo)
linhaLocalidade: 3, // linha 3 = localidade por coluna

linhasTrocaTurno: [34, 35, 36], // linhas Excel das TTs
⚠️
linhaCarroID, linhaSentido e linhaLocalidade são números de linha relativa dentro da grade, não linha do Excel. Linha 1 da grade = primeira linha do intervalo definido em intervaloGeral.

Engine — Funcionamento Interno

Detalhamento do fluxo de execução de Engine.process(workbook).

1

Leitura da Planilha

SheetJS converte a aba em uma matriz 2D (sheet_to_json header:1, raw:true). Células de hora chegam como serial decimal ou objeto Date — o método toMin() normaliza qualquer formato para minutos inteiros.

2

Configuração Global

readGlobalConfig() lê as células F2–F11 e resolve o layoutConf a usar. O resultado (globalCtx) é passado a todos os construtores de linha.

3

Leitura de Exceções

readExcecoesViagens() gera um array de regras { criterios, overrides }. readExcecoesTabelas() gera um Map de tabKey → overrides. Ambas são lidas uma única vez antes do loop de carros.

4

Loop de Carros

O engine itera de coluna em coluna com passo totalColunas. Para cada bloco: lê o ID do carro, resolve localidades por coluna, chama buildTrips().

5

buildTrips()

Percorre as linhas da grade e monta viagens brutas { dir, start, end, hasEnd, row }. A lógica de fim de viagem segue os slots do layout. hasEnd=false sinaliza intervalo ou fim de serviço.

6

readTrocasTurno()

Lê as linhas de TT do carro e valida cada horário contra o validTripsSet. Apenas TTs que correspondam a viagens reais são aceitos.

7

splitIntoTables()

Divide o array de viagens em tabelas. Cortes ocorrem em dois eventos: (a) viagem com start+dir igual ao próximo TT pendente; (b) viagem com hasEnd=false quando há viagens subsequentes.

8

Nomeação e buildTableRows()

Cada tabela recebe uma letra via nextLetter(). buildTableRows() aplica exceções de tabela, calcula turn/preparo/acesso/recolhe, emite linhas produtivas e a linha de encerramento (TT / intervalo / recolhe).

Resolução de Turno

resolveturno(tabStartMin) percorre mapaTurnos ordenado e retorna o turno do primeiro corte maior que tabStartMin. Se houver override via exceção de tabela (periodo), esse valor prevalece.

Resolução de Local de Encerramento

Para TT, intervalo e recolhe, o local é resolvido pela seguinte prioridade:

1. Override da exceção de viagem (campo F)
2. settings.atividades[chave].local (ex: intervalo → '07')
3. codLocalIda ou codLocalVolta conforme o sentido

Virada de Dia

O método isAfter(a, b) detecta virada de dia: se a - b > 12*60, assume que b pertence ao dia seguinte. Isso garante que horários após meia-noite (ex: 00:06) sejam tratados corretamente como posteriores a 23:50.

Exporter — Formatação e Saída

Exporter é uma camada puramente de formatação. Não há lógica de negócio aqui.

formatField(value, conf)

Normaliza um valor para string, aplica truncamento de hora (type: 'hour'), faz padding conforme align e garante o tamanho exato com substring(0, size).

toTXT(data)

Para cada contexto, chama resolveRow() (que resolve e formata todos os campos) e concatena sem separador. Linhas separadas por \r\n.

toCSV(data)

Primeira linha com os nomes dos campos (field). Demais linhas com valores separados por ;. Se aplicarFormatacaoNoCsv for false, os valores são emitidos sem padding.

download(content, filename, mime)

Cria um Blob, gera uma URL temporária, simula um clique em um <a> e revoga a URL após o download. Compatível com todos os navegadores modernos.

⚙️
Para adicionar um novo campo na saída, basta inserir uma nova entrada no array SETTINGS.layout com sua função resolve. O Exporter percorre o array automaticamente — nenhuma alteração no export.js é necessária.
